# Tidy Up — Godot project

**Tidy Up** is a cozy *merge-and-tidy* game: combine clutter, clear the room, relax.
Drag any two matching items together to merge them up a tier ladder, complete jobs for
the neighbourhood, and restore a hand-painted "Ghibli Grove" homestead.

The canonical design — spec **and** feature index — lives in
[`docs/core/merge_spec.md`](docs/core/merge_spec.md). (This project began as
"Reach Zero," a merge-solitaire prototype — that original spec and the pivot history are
kept under [`docs/archive/`](docs/archive/).)

## Requirements
Godot **4.6** (developed on 4.6.2). On this machine: `/opt/homebrew/bin/godot`.

## Run the game
- **Editor:** open this folder as a Godot project and press Play.
- **CLI (desktop):** `godot --path .` — opens a portrait window; mouse-click acts as touch.
- Main scene: `scenes/Home.tscn`.

## Run the tests (headless — no display needed)
```bash
make test           # every suite (engine + game)
make test-engine    # base-engine suites only
make test-grove     # grove game suites only
make test-one SUITE=engine/tests/save_tests   # a single suite
make smoke          # scene-instantiation smoke check
```
Suites live beside the code they cover: the base-engine ones in `engine/tests/`
(`save_tests`, `layout_tests`, `mechanics_tests`, `layering_tests`, plus the scene
`smoke`), and the grove game's in `games/grove/tests/` (`grove_tests`).

## Layout
| Path | Role |
|---|---|
| `scripts/` | Game logic. `board.gd` is the pure, verified rules engine (no UI deps); `grove.gd` / `home.gd` drive the live loop. |
| `scenes/` | `Home.tscn` (hub / main scene), `Grove.tscn`, `Room.tscn`, `Menu.tscn`. |
| `assets/` | Source art & audio — hand-painted grove style. |
| `data/` | Game data. Tests + dev tools live beside the code they serve: `engine/{tests,tools}`, `games/tools`, `games/<game>/{tests,tools}`. |
| `docs/` | Design specs, art pipeline, and planning. `docs/archive/` holds superseded history. |
| `build/` | iOS export output — **gitignored**, regenerated by `godot --export`. |

## Mobile / iOS
Portrait (`display/window/handheld/orientation=1`) with the mobile renderer. iOS build &
sideload use the `export_presets.cfg` "iOS" preset; exports land in the
gitignored `build/ios/`.
