# Acorn Forest: Merge! — Godot project

**Acorn Forest: Merge!** is a cozy *merge-to-grow* game: a child crosses into a fading
spirit-forest and tends one persistent merge board. Drag any two matching items together
to merge them up a tier ladder, deliver harvests to the grove's spirit-folk for Stars,
and restore a hand-painted "Ghibli Grove" homestead.

The canonical design — spec **and** feature index — lives in
[`docs/design/merge_spec.md`](docs/design/merge_spec.md). (Early prototype specs and the pivot history are kept under
[`docs/archive/`](docs/archive/).)

## Requirements
Godot **4.6** (developed on 4.6.2). On this machine: `/opt/homebrew/bin/godot`.

## Run the game
- **Editor:** open this folder as a Godot project and press Play.
- **CLI (desktop):** `godot --path .` — opens a portrait window; mouse-click acts as touch.
- Main scene: `scenes/Home.tscn`.

## Run the tests (headless — no display needed)
Suites run **in parallel** with a per-suite timing table (`engine/tools/run_suites.py`,
`JOBS=4` by default). After **every change**, run the fast inner-loop check; run the
full sweep only before you commit or hand off:
```bash
make test-fast      # ⚡ INNER LOOP — engine suites only, parallel (use after every change)
make test           # full sweep: every suite (engine + grove), parallel + timing table
make test-engine    # base-engine suites only
make test-grove     # grove game suites only
make test-one SUITE=engine/tests/save_tests   # a single suite (serial)
make smoke          # scene-instantiation smoke check
JOBS=8 make test    # raise/lower parallelism
```
Suites live beside the code they cover: the base-engine ones in `engine/tests/`, and the
grove game's in `games/grove/tests/`. The grove suite was split from one 2.3k-line
monolith into focused suites (`grove_model_tests`, `grove_economy_tests`, `grove_ui_tests`,
`grove_placement_tests`, `grove_shop_ads_tests`) sharing `grove_test_base.gd`, so they
parallelise and you can run just the slice you touched.

## Layout
| Path | Role |
|---|---|
| `scripts/` | Game logic. `board.gd` is the pure, verified rules engine (no UI deps); `grove.gd` / `home.gd` drive the live loop. |
| `scenes/` | `Home.tscn` (hub / main scene), `Grove.tscn`, `Room.tscn`, `Menu.tscn`. |
| `assets/` | Source art & audio — hand-painted grove style. |
| `data/` | Game data. Tests + dev tools live beside the code they serve: `engine/{tests,tools}`, `games/tools`, `games/<game>/{tests,tools}`. |
| `docs/` | Design specs, art pipeline, and planning. `docs/archive/` holds superseded history. |
| `build/` | iOS export output — **gitignored**, regenerated by `godot --export`. |

## Mobile / iOS
Portrait (`display/window/handheld/orientation=1`) with the mobile renderer. iOS build &
sideload use the `export_presets.cfg` "iOS" preset; exports land in the
gitignored `build/ios/`.
